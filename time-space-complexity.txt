- big o notation: it is way to measure time and space complexity:
1. o(1)-  constant time (means number of input increase but number of operation is same)
2. o(n)- linear time (number of operation increases as the number of input increases)
for ex: for(int i=0;i<n;i++){
    cout<<i;
}..here if n=5 then loop will run 5 time if n=10 then loop will run 10


3. 0(n square)= nested loop
4. 0(log n)= every step data half

//space complexity: here we find how many extra space/memory our code is using excluding input:
for ex:
int sum =0;
for(int i=0;i<n;i++)
{
    sum += arr[i];
}
- extra is sum which is one so sc=o(1) constant

vector<int> temp;

for(int i=0;i<n;i++)
{
    temp.push_back(arr[i]);
}
//if n=5 then temp will be 5 so here the o(n)
 when calculating space complexity, we do not count the input itself.

The array is already provided as input.

We only consider how much additional (extra) memory the algorithm uses besides the input.

Therefore, in the following code:

int sum = 0;

only a constant amount of extra memory is used (just one variable).

So, the space complexity is O(1).
